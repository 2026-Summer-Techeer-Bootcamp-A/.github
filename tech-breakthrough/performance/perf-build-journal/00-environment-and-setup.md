# 환경과 첫 설정: 있는 것을 확인하고 안전한 범위를 정한다

## 목표

부하 테스트를 시작하기 전에 두 가지를 먼저 확인했다. 이미 갖춰진 관측 도구가 무엇인지, 그리고 어디까지 세게 테스트해도 안전한지다. 새로 뭔가를 만들기 전에 있는 것을 확인하는 순서를 지켰다.

## 이미 있던 것 발견

Grafana 대시보드 목록을 살펴보다가 `FastAPI Backend, Load & Performance`라는 이름의 대시보드를 발견했다. 패널을 열어보니 처리량, p50/p90/p95/p99 지연<sup class="fnref" id="fnref1"><a href="#fn1">1</a></sup>, 오류율, Apdex 점수<sup class="fnref" id="fnref2"><a href="#fn2">2</a></sup>, 엔드포인트별 순위까지 27개 패널이 이미 만들어져 있었다.

이 대시보드가 참조하는 데이터를 추적해보니, API 서버가 이미 모든 요청의 처리량과 지연을 자체적으로 계측해서 노출하고 있었고, 시계열 데이터베이스가 5초 주기로 이 값을 가져가고 있었다. 즉 부하 테스트 도구를 새로 연결할 때 서버 쪽에 지표를 보내는 코드를 따로 짤 필요가 없다는 뜻이었다. 부하 테스트 도구는 트래픽만 만들면, 이미 있는 파이프라인이 그 결과를 자동으로 그려준다.

이 발견은 작업 범위를 크게 줄였다. 계측 코드를 새로 짜는 대신, 트래픽 생성 스크립트만 작성하고 기존 대시보드를 보는 방법을 문서로 남기면 되는 상황이었다.

## 도구 선정: k6

스크립트를 자바스크립트로 짜고, 가상 사용자 수와 지속 시간을 단계로 정의하고, 통과 기준을 코드에 명시할 수 있는 도구가 필요했다. k6<sup class="fnref" id="fnref3"><a href="#fn3">3</a></sup>를 선택했다.

## 절대 건드리지 않을 경로 확정

트래픽을 만들기 전에, 어떤 엔드포인트를 때리면 안 되는지를 먼저 정리했다.

첫째는 챗봇 엔드포인트다. 이 서비스의 RAG 챗봇은 질문마다 외부 LLM API를 호출하는데, 부하 테스트로 반복 요청을 보내면 그대로 실제 비용이 나간다. 성능을 재려는 목적과 무관하게 돈이 드는 경로라서, 대상에서 완전히 제외했다.

둘째는 이력서 파싱 엔드포인트다. 파일 처리와 LLM 피드백을 포함해서 무겁고, 세션 상태를 만들기 때문에 반복 호출이 부담스럽다.

셋째는 모든 쓰기 엔드포인트다. POST나 PUT, DELETE로 데이터를 만들거나 바꾸는 요청은 반복 실행하면 더미 데이터가 쌓이거나 실제 데이터가 오염된다.

그래서 첫 스크립트는 조회만 하는 GET 엔드포인트 네 개로 좁혔다. 헬스체크, 공고 목록, 기술 스택 목록, 직군 목록이다.

## 프로덕션 사양 확인

이 서비스가 도는 가상머신은 2 vCPU에 8GB 메모리이고, 그 안에서 API 서버뿐 아니라 리버스 프록시와 관측 스택 전체가 함께 돈다. 데이터베이스는 최근 비용 절감을 위해 1 vCPU에 3.75GB로 사양을 낮춘 상태였다. 이 사실을 확인한 뒤, 프로덕션에 거는 부하는 가상 사용자 수와 지속 시간을 로컬보다 훨씬 작게 잡기로 정했다. 작은 사양에 큰 부하를 걸면 성능을 재려다가 서비스 자체에 장애를 일으킬 수 있기 때문이다.

## 파일 배치 위치 결정

처음에는 백엔드 레포 안에 `loadtest` 폴더를 만들어 스크립트를 넣었다. 그런데 이 작업은 RAG 작업 때 `rag-development` 폴더를 따로 만들어 개념과 구현 과정을 문서로 쌓았던 것과 같은 성격의 작업이었다. 그래서 같은 방식으로 저장소 최상위에 `performance-test` 폴더를 만들고, 그 안에 개념서 폴더와 구현기 폴더, 그리고 실제 k6 스크립트를 담는 폴더를 나눠 배치했다. 문서를 읽기 좋게 보여주는 웹 뷰어도 `rag-development`의 것을 그대로 복사해 이름만 바꿔 재사용했다.

## 남은 것

k6 자체는 아직 로컬에 설치하지 않았다. 다음 단계는 k6를 설치하고, 로컬 개발 스택을 띄운 상태에서 스모크 테스트부터 돌려 스크립트가 의도대로 동작하는지 확인하는 것이다. 그 결과와 실제 수치는 이어지는 문서에 채운다.
